- When a kit already exists, follow its implementation convention instead of regenerating from scratch — if existing kit components wrap an installed component library, new/extended ones must compose that same library; the self-contained "framework-primitives-only" rule is the greenfield default, applied only when there is no existing kit or library convention.
- Every UI spec must include shell/navigation integration for each screen: how the user reaches it from the existing shell (entry) and where each primary CTA leads (exit) — never spec a chrome-less screen for an app that already has a global shell.
- When carrying the design-system memory forward, preserve human-pinned standing rules verbatim and only ADD to them — compacting the design-system doc must never evict a rule a human pinned.
